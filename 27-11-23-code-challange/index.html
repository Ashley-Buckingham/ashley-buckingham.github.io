2722. Join Two Arrays by ID

Introduction

Hello, coding enthusiasts! We're about to embark on an interesting LeetCode challenge today: Join Two Arrays by ID (Challenge 2722). This task might appear simple at first, but it's an excellent platform to delve into some of the more subtle aspects of JavaScript and discover some nifty optimization strategies. Let's take a journey through my coding process and insights!

The Problem

Our task involves two arrays, arr1 and arr2, each containing objects with an id field. We aim to merge them into a single array, joinedArray, following these guidelines:

  • Merge objects using their id as the key.
  • If an id is found in both arrays, blend their properties, with arr2 taking precedence over arr1.
  • Finally, sort joinedArray in ascending order based on the id.

Initial Idea

To tackle this, I started by conceptualizing the function's structure. My plan was to iterate through arr2, check for the presence of each element in arr1, and perform one of two actions:

  • If an element from arr2 didn't exist in arr1, I'd simply add it to joinedArray.
  • If it did exist, I'd merge their properties, ensuring arr2's values override those from arr1.
  • The last step would be sorting joinedArray based on id.

Locating Elements

Initially, I leaned towards using lodash but decided to stick to native JavaScript functions for a cleaner approach. I used forEach to traverse through arr2 and utilized the .find method to check if an element already exists in arr1.

However, I realized I needed to know the specific index of the found element in joinedArray. Therefore, I replaced .find with .findIndex.

Conclusion

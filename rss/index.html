The Problem

Our task involves two arrays, arr1 and arr2, each containing objects with an id field. We aim to merge them into a single array, joinedArray, following these guidelines:

  • Merge objects using their id as the key.
  • If an id is found in both arrays, blend their properties, with arr2 taking precedence over arr1.
  • Finally, sort joinedArray in ascending order based on the id.

Initial Idea

To tackle this, I started by conceptualizing the function's structure. My plan was to iterate through arr2, check for the presence of each element in arr1, and perform one of two actions:

  • If an element from arr2 didn't exist in arr1, I'd simply add it to joinedArray.
  • If it did exist, I'd merge their properties, ensuring arr2's values override those from arr1.
  • The last step would be sorting joinedArray based on id.

Locating Elements

Initially, I leaned towards using lodash but decided to stick to native JavaScript functions for a cleaner approach. I used forEach to traverse through arr2 and utilized the .find method to check if an element already exists in arr1.

However, I realized I needed to know the specific index of the found element in joinedArray. Therefore, I replaced .find with .findIndex.

DEMO

The demo below shows how I can check stock without going through the website steps. Entering a part ID and a postcode will check real-time inventory in your local stores. A simple notification system can be added to the code to alert users when a particular store is in stock.

Explanation

The code for this project is simple. The process for understanding the request is even easier. To understand what type of request i could make i went to apple.com selected a product and clicked 'pickup', I was presented with the following screen.

Searching with a postcode sends a request to Apple's servers to return the stock levels of the given product.
Using the web browser's developer tools, you can monitor the network traffic as you click the search button.

Here we can see a GET request to apple.com with a few parameters. Using what we know about Apple, reconstructing this URL is very easy.

https://www.apple.com/uk/shop/fulfillment-messages?searchNearby=true&parts.0=MLY43B/A&location=se1%201aa

becomes

https://www.apple.com/uk/shop/fulfillment-messages?searchNearby=true&parts.0=<part_number>&location=<post_code

Now let's look at the response to the request(below).
The server returns many fields; I have removed many of those to increase readability. The interesting parts of the JSON are "storeName", "partsAvailability", and "storePickEligible". These fields show which stores have stock.

Using the information we know about the request, we can create a simple node.js script to get this data. Then, a developer can edit the code to add a notification system that will fire when product stock appears; we have successfully used Slack to buy many products.
